## 🔧 Google Apps Script 部署步驟

### 步驟 1: 創建 Google 試算表
1. 進入 [Google Sheets](https://sheets.google.com)
2. 新建試算表
3. 命名為「多國貨幣記帳」
4. 複製 URL 中的試算表 ID

**試算表 URL 示例：**
```
https://docs.google.com/spreadsheets/d/1ABC...XYZ/edit
                                          ↑ 複製這部分
```

### 步驟 2: 打開 Google Apps Script
1. 在試算表菜單選擇「擴充功能」
2. 點擊「Apps Script」
3. 新視窗打開 Google Apps Script 編輯器

### 步驟 3: 添加代碼
1. 刪除預設的空白代碼
2. 複製 `gas-code.js` 的所有內容
3. 粘貼到 Google Apps Script 編輯器

### 步驟 4: 配置試算表 ID
編輯第 11-12 行：
```javascript
const SHEET_ID = 'YOUR_SPREADSHEET_ID'; // 改為你的ID
const SHEET_NAME = '消費記錄';          // 保持不變
```

### 步驟 5: 部署為 Web App

**5.1 部署設置**
1. 點擊頂部「部署」按鈕
2. 選擇「新增部署」
3. 點擊「選擇類型」→ 「Web 應用程式」

**5.2 部署配置**
- **執行身份**：選擇你的 Google 帳號
- **有權存取的使用者**：選擇「所有人」

**5.3 完成部署**
1. 點擊「部署」
2. 出現授權提示，點擊「授權此應用程式」
3. 複製部署後的 URL (格式如下)

**部署 URL 格式：**
```
https://script.google.com/macros/s/XXXXXXXXXX/usercopy
```

### 步驟 6: 更新前端配置
1. 打開 `script.js`
2. 找到第 8 行 `const GAS_DEPLOYMENT_URL`
3. 替換為你的部署 URL

```javascript
const GAS_DEPLOYMENT_URL = 'https://script.google.com/macros/s/YOUR_ID/usercopy';
```

## 🧪 完整流程測試

### 測試場景 1: 本地存儲 (無 GAS)

1. **初始化**
   - 打開 `index.html`
   - 打開瀏覽器 Developer Tools (F12)
   - 進入 Console 標籤

2. **新增消費**
   - 填寫「消費項目」：機票
   - 選擇「幣別」：USD
   - 輸入「金額」：1000

3. **自動匯率**
   - 點擊「自動獲取匯率換算」
   - Console 應顯示 API 調用
   - 應填入「即時匯率」和「折合台幣」

4. **儲存**
   - 點擊「儲存消費」
   - 應看到「成功」提示

5. **驗證本地存儲**
   - 在 Console 執行：
   ```javascript
   localStorage.getItem('expenses')
   ```
   - 應返回 JSON 字符串，包含上述數據

### 測試場景 2: Google 試算表同步

1. **前置條件**
   - GAS 已部署
   - `script.js` 已更新 URL

2. **新增消費**
   - 按照測試場景 1 的步驟新增消費

3. **驗證同步**
   - 打開 Google 試算表
   - 應在「消費記錄」工作表看到新數據

4. **驗證計算**
   ```
   台幣金額 = 外幣金額 × 匯率
   例: 1000 USD × 30.5 = 30,500 TWD
   ```

### 測試場景 3: 預算進度

1. **初始預算**
   - 預算設置為 10,000 TWD
   - 進度條應為空

2. **新增消費 1**
   - 新增 500 CNY (約 2,250 TWD)
   - 進度條應填充 22.5%

3. **新增消費 2**
   - 新增 200 EUR (約 6,800 TWD)
   - 進度條應填充 92.5%（變黃）

4. **新增消費 3**
   - 新增 100 JPY (約 200 TWD)
   - 進度條應 100%（變紅）

### 測試場景 4: 側邊欄功能

1. **打開側邊欄**
   - 切換到「歷史流水帳」
   - 點擊任何消費記錄的「查看詳細」

2. **驗證側邊欄**
   - 應從右側平滑滑出
   - 應顯示完整的消費信息
   - 應有「刪除」按鈕

3. **測試刪除**
   - 點擊「刪除」按鈕
   - 應要求確認
   - 刪除後側邊欄應關閉

### 測試場景 5: 動畫效果

1. **分頁切換動畫**
   - 在「新增消費」和「歷史流水帳」之間切換
   - 內容應平滑淡入淡出（0.6秒）

2. **側邊欄動畫**
   - 點擊消費記錄
   - 側邊欄應平滑滑出
   - 背景應半透明

3. **預算條動畫**
   - 新增消費時
   - 進度條應平滑伸展（0.5秒）

## ⚠️ 常見錯誤

### 錯誤 1: "Cannot read property 'postData'"
- **原因**：GAS doPost 函數配置錯誤
- **解決**：檢查 `gas-code.js` 代碼是否完整

### 錯誤 2: "API 調用失敗"
- **原因**：網絡問題或 Frankfurter API 不可用
- **解決**：檢查網絡連接，稍後重試

### 錯誤 3: "CORS 錯誤"
- **原因**：跨域請求被阻擋
- **解決**：已在 `script.js` 中使用 `mode: 'no-cors'` 解決

### 錯誤 4: "試算表不存在"
- **原因**：SHEET_ID 配置錯誤
- **解決**：重新檢查試算表 ID
