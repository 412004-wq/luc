# 👋 使用教學 - 詳細步驟指南

## 🎬 30 秒快速示範

### 場景：記錄東京出差的消費

```
1. 打開 index.html
2. 輸入「機票」→ 選擇「JPY」→ 輸入「150000」
3. 點擊「🔄 自動獲取匯率換算」
4. 系統自動計算：150000 JPY × 0.222 = 33,300 TWD
5. 點擊「💾 儲存消費」✓
6. 切換到「歷史流水帳」查看記錄
```

## 📖 完整教學

### 第 1 步：打開應用

#### 方式 A：直接打開（推薦）
1. 找到 `index.html` 文件
2. 雙擊打開
3. 瀏覽器會自動加載頁面

#### 方式 B：VS Code 打開
1. 用 VS Code 打開整個 `luc` 文件夾
2. 右鍵點擊 `index.html`
3. 選擇「Open with Live Server」
4. 頁面自動在瀏覽器打開

#### 方式 C：上傳到網站
1. 上傳 `index.html`、`styles.css`、`script.js` 到服務器
2. 在瀏覽器訪問網站 URL

### 第 3 步：新增第一筆消費

#### 3.1 填寫基本信息

1. **消費項目欄**
   - 點擊「消費項目」輸入框
   - 輸入消費內容（例：機票、飯店、購物）
   
   💡 **提示**：描述越詳細越好，便於日後查找

2. **選擇幣別**
   - 點擊「幣別」下拉選單
   - 選擇對應幣別
   - 支持：USD、JPY、EUR、KRW
   
   ```
   🇺🇸 USD - 美國出差
   🇯🇵 JPY - 日本出差
   🇪🇺 EUR - 歐洲出差
   🇰🇷 KRW - 韓國出差
   ```

3. **輸入外幣金額**
   - 點擊「金額」輸入框
   - 輸入你的消費金額（支持小數點）
   - 例：100、1500.50、999.99

#### 3.2 自動匯率換算

1. **點擊「🔄 自動獲取匯率換算」按鈕**
   - 系統會自動調用匯率 API
   - 按鈕會顯示加載中的狀態

2. **等待系統計算**
   - 通常需要 1-3 秒
   - 成功時會看到綠色提示：「✓ 成功取得匯率」
   - 失敗時會看到紅色提示：「❌ 無法獲取匯率」

3. **檢查自動填入的數據**
   ```
   即時匯率：30.5000
   折合台幣：$ 3,050.00
   ```

#### 3.3 添加備註（可選）

1. 點擊「消費備註」文本框
2. 輸入額外信息（例：會議費、接待費等）
3. 備註會一起保存到記錄中

#### 3.4 保存消費

1. 點擊「💾 儲存消費」按鈕
2. 系統會：
   - 驗證所有必填欄位
   - 保存到本地存儲
   - 發送到 Google 試算表（如已配置）
   - 顯示成功提示
3. 表單自動清空，準備新輸入

### 第 6 步：管理預算

#### 6.1 了解預算條

頁面頂部的預算進度條會：

- **實時更新**：每次新增消費都會更新
- **顏色變化**：
  - 🟠 橙色：0-70% 的預算
  - 🟡 黃色：70-90% 的預算
  - 🔴 紅色：超過 90% 的預算

#### 6.2 查看預算信息

```
本月預算進度
[████████░░░░] 35%
5,300 / 15,000 TWD
```

這表示：
- 已支出：5,300 元
- 預算上限：15,000 元
- 使用比例：35%

#### 6.3 修改預算上限（可選）

編輯 `script.js` 第 13 行：

```javascript
const MONTHLY_BUDGET = 20000;  // 改為你的預算
```

**遺憾地，目前版本不支持編輯。解決方法：**

1. 進入歷史流水帳
2. 找到錯誤的消費
3. 點擊「查看詳細」
4. 點擊「🗑️ 刪除」刪除
5. 重新新增正確的消費

#### 方式 1：從 Google 試算表導出（如已同步）
1. 打開 Google 試算表
2. 選擇所有數據
3. 右鍵 → 下載 → CSV 或 Excel

#### 方式 2：從瀏覽器導出
1. 按 F12 打開開發者工具
2. 進入 Console 標籤
3. 輸入：
   ```javascript
   console.log(localStorage.getItem('expenses'))
   ```
4. 複製輸出的 JSON 數據
5. 粘貼到文本編輯器或 Excel

## ⚠️ 常見問題在使用中

### 問題 1：「自動獲取匯率」沒反應

**原因和解決**：
- ❌ 未選擇幣別 → 先選擇幣別
- ❌ 未輸入金額 → 先輸入金額
- ❌ 網絡問題 → 檢查網絡連接
- ❌ API 不可用 → 稍後重試

### 問題 2：儲存後消費沒有出現

**檢查步驟**：
1. 切換到「歷史流水帳」分頁
2. 如果看不到，檢查：
   - 瀏覽器 Console (F12) 是否有錯誤
   - 所有欄位是否都填了
   - 匯率是否成功獲取

### 問題 3：側邊欄沒有滑出

**原因**：
- JavaScript 可能未加載
- 瀏覽器不支持
- 頁面需要刷新

**解決**：
1. F5 刷新頁面
2. 重新點擊消費記錄

### 問題 4：數據丟失

**保護數據**：
- 瀏覽器本地存儲通常安全
- 定期導出備份
- 啟用 Google 試算表同步（永久保存）

## 🎓 進階操作

### 進階 1：批量數據操作

在瀏覽器 Console 運行：

```javascript
// 查看所有消費
console.table(JSON.parse(localStorage.getItem('expenses')))

// 計算本月總支出
const exps = JSON.parse(localStorage.getItem('expenses'))
const total = exps.reduce((sum, e) => sum + e.twdAmount, 0)
console.log('本月支出：' + total)

// 清空所有數據（謹慎！）
localStorage.clear()
```

### 進階 2：自定義預算

```javascript
// 按季度調整
Q1: 30000  // 1-3月

```

3. 實現多設備共享記帳本
